# 已部署平台清理指南

## 🎯 目标
清理不再使用的部署资源，避免占用免费额度和产生潜在费用。

## 1️⃣ Vercel 清理（已迁移到 Cloudflare）

### 步骤 1: 删除 Vercel 项目

#### 通过 Web Dashboard（推荐）
1. 访问 https://vercel.com/dashboard
2. 找到项目 `gaoshijiazu`（或类似名称）
3. 点击项目进入详情
4. 点击 **Settings** → **General**
5. 滚动到底部，点击 **Delete Project**
6. 输入项目名称确认删除
7. ✅ 完成

#### 通过 Vercel CLI（可选）
```bash
# 安装 Vercel CLI（如果未安装）
npm install -g vercel

# 登录
vercel login

# 列出所有项目
vercel list

# 删除项目（替换 PROJECT_NAME）
vercel remove gaoshijiazu
```

### 步骤 2: 清理本地 Vercel 文件
```bash
cd "D:/aibiancheng/aixiangmu/gaoshijiazu1.0"

# 删除 .vercel 目录
rm -rf .vercel

# 删除 vercel.json 配置文件（可选）
rm vercel.json

# 提交删除
git add -A
git commit -m "清理 Vercel 部署配置"
```

### 步骤 3: 断开 GitHub 连接
如果 Vercel 连接了 GitHub：
1. 访问 https://github.com/settings/installations
2. 找到 **Vercel** 应用
3. 点击 **Configure**
4. 找到 `gaoshijiazu` 仓库
5. 点击 **Uninstall** 或取消授权

## 💡 为什么清理？

### 1. 避免资源浪费
- Vercel 免费额度有限（100GB/月）
- Railway 可能产生费用
- 未使用的项目占用名额

### 2. 安全考虑
- 避免遗留部署版本被误访问
- 减少安全风险暴露面

### 3. 项目整洁
- 只保留实际使用的平台配置
- 减少维护复杂度
