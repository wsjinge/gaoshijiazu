## 📍 三个操作位置（按推荐顺序）

### 1️⃣ **左侧源代码管理面板** ⭐ 最推荐（可视化）

#### 如何找到
```
左侧边栏 → 点击"源代码管理"图标（🌿 分支形状）
或快捷键: Ctrl + Shift + G
```

#### 操作步骤
```
1. 打开左侧"源代码管理"面板
2. 在文件列表上方找到 "..." 更多操作菜单
3. 点击 → 选择"推送"
   或: "推送到..." → 选择 "origin/master"
```

#### 界面示意
```

│ 源代码管理      │
│ ─────────────   │
│ 📝 更改 (0)     │ ← 未提交修改

│ ✅ 已暂存更改   │ ← 待提交文件
│ ─────────────   │
│ [提交按钮]      │
│ [⋮ 更多操作]    │ ← 点击这里找推送
│   ├── 推送 ✅   │ ← 你要找的功能
│   ├── 拉取      │
│   ├── 同步      │
│   └── ...       │
└─────────────────┘
```

### 3️⃣ **Claude 插件对话** 🤖 最智能（自动执行）

#### 如何操作
```
直接在 Claude 对话框中说：
"推送到 GitHub"
或: "git push"
```

#### Claude 会自动
```
1. 检查当前状态 (git status)
2. 添加修改 (git add，如果需要)
3. 提交修改 (git commit，如果需要)
4. 推送到 GitHub (git push)
5. 报告推送结果
```

#### 示例对话
```
你: "推送到 GitHub"

Claude: 好的，我来帮你推送...
        [执行 git status]
        [执行 git push origin master]
        ✅ 推送成功！
        GitHub 已更新到最新版本。
```

## 🔄 Git 工作循环（在 Trae CN 中）

### 开发 → 提交 → 推送
```
1. 编辑代码（编辑器窗口）
   ↓
2. 保存文件（Ctrl + S）
   ↓
3. 查看修改（左侧源代码管理 → 更改）
   ↓
4. 暂存修改（点击文件旁的 "+" 图标）
   ↓
5. 提交修改（输入提交信息 → 点击"提交"）
   ↓
6. 推送到 GitHub（点击 "⋮" → "推送"）
   ↓
7. ✅ GitHub 更新完成
```
